# ポートフォリオサイト要件定義

## ゴール
- 作成したプロジェクト一覧と、書いた記事一覧を一目で確認できる。
- 公開済み状態で誰でもアクセスできる (Vercelでホスティング)。
- 代表作と記事にスムーズに移動できる導線を用意する。

## スコープ
- **MVP (初版)**: ホーム、プロジェクト一覧/詳細、スキル概要、プロフィール、コンタクト(フォーム or メールリンク)。
- **Nice to have**: ブログ/ノート、英語切り替え、ダウンロード可能なPDFレジュメ、ダーク/ライト自動切替。

## 情報設計
- **ホーム**: キャッチコピー、主要実績3件への導線、最新記事 or 更新情報。
- **プロジェクト一覧**: サムネイル・タグ・担当領域・使用技術をカード表示、フィルタ(領域/技術/年)。
- **プロジェクト詳細**: 課題・アプローチ・成果・担当範囲・使用スタック・スクリーンショット・成果リンク。
- **スキル/サービス**: 言語・フレームワーク・設計/DevOps・デザインツールをレベル感付きで列挙。
- **プロフィール**: 経歴タイムライン、価値観/強み、メディア掲載や登壇があれば掲載。
- **コンタクト**: フォーム(スパム防止), SNS/LinkedIn/GitHub へのリンク、応答SLAの明記。

## 機能要件
- ナビゲーション: PC/モバイル共通ヘッダー、フッターに主要リンクとSNS。
- 下部固定タブ: 「ポートフォリオ」「ブログ」「スキル」「お問い合わせ」の4タブで各セクションへスクロール遷移できるようにする。
- 検索/フィルタ: プロジェクト一覧でタグフィルタ。MVPでは静的フィルタのみで可。
- フォーム: メール送信 or 外部フォーム(例: Formspree)で実装、バリデーションと送信結果表示。
- 画像: 最適化されたレスポンシブ画像、サムネイル比率を統一。
- 国際化: MVPは日本語。将来英語追加を想定し構造を i18n 対応しやすく。
- メタ情報: OGP / SEO メタタグ、構造化データ(JSON-LD)でプロフィールと作品をマークアップ。

## 非機能要件
- パフォーマンス: Lighthouse Performance/Best Practices/SEO 90点以上、Largest Contentful Paint 2.5s 未満を目標。
- アクセシビリティ: Lighthouse Accessibility 90点以上、キーボード操作/フォーカススタイル/aria-label整備。
- レスポンシブ: 360px〜1440px で主要コンポーネントが崩れない。
- 運用性: CMS 不要で md/JSON からコンテンツを差し替え可能にするか、Headless CMS 導入を検討(後続タスク)。
- セキュリティ: 外部フォーム利用時は reCAPTCHA などのスパム対策、依存パッケージを毎月更新チェック。

## コンテンツ要件
- 代表作: 3〜6件。各件に「背景/課題」「目標」「担当」「成果」「技術」「期間」「リンク」を記載。
- スクリーンショット: 各プロジェクトに最低1枚。縦横比 4:3 推奨。
- スキルセット: 言語・FW(Next.js 16/React 19/Tailwind CSS 4)・インフラ/設計・デザインツールを分類。
- 自己紹介: 100〜150字のショートバイオ＋詳細版。顔写真またはロゴを1点。
- コンタクト: 希望する相談内容の例(制作/改善/スポット相談など)を明記。

## デザイン指針
- トーン: シンプルで誠実、モノトーン＋1アクセントカラー(ブルー系を想定)を基調。
- タイポグラフィ: 見出しはサンセリフ太め、本文は可読性重視。Webフォント1種＋システムフォント fallback。
- レイアウト: セクション幅は 1200px を上限、カードは2〜3カラムのグリッド、ホバーとモーションは最小限で質感を出す。

## 技術スタック方針
- Next.js 16 (App Router) + React 19 + TypeScript。
- スタイリング: Tailwind CSS v4 をベースに、テーマ変数を `tailwind.config` または `globals.css` で集中管理。
- データ管理: MVPでは静的データ(JSON/MD)。将来的にCMS導入を見越し、データ取得を抽象化する層を用意する。
- デプロイ: Vercel を想定。プレビュー環境をブランチごとに自動生成。
- テスト: Playwright で主要導線の E2E、Lighthouse CI でパフォーマンス/アクセシビリティ継続計測(後続タスク)。

## 測定/KPI
- 主要ボタン(プロジェクト閲覧/コンタクト送信)のクリック率と完了率を Vercel Analytics などで計測。
- 月1回の更新サイクルを維持するため、更新しやすいデータ構造かを運用レビューで確認。
